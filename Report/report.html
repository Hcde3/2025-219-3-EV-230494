Plan and Design

Chosen Idea

The idea I selected is exoplanets because I have a lot of interest and prior knowledge in the topic.

My target audience for my information system is astrophysicists and astronomy enthusiasts above the age of 12.

My information system aims to show the pattern and relation of variables between different exoplanet types providing a general analysis and intersting information about these celestial bodies to my target audience.

I will be using the Waterfall method of software development to plan and create my project. I will also be aware and conscious of ethical considerations especially in the user form. While personal information is collected, the form is completely voluntary and collects no more data than name and age.

Meeting Brief Requirements

My program will meet the basic requirements by creating a python program to extract data from my dataset, clean it and store it in a database.

A second Python program will read the clean data from my database and analyse it in a variety of ways including by categorising the exoplanets and finding averages. I will display at least 2 visualisations in the form of basic graphs and charts all with labels and titles. All my analysis will be sent to my database.

I will create a web interface which displays the data from my analysis.

My program will meet the advanced requirements by making the webpage graphs interactive, and I will have a section at the bottom of my main page which will help inform users further about particular aspects of my topic.

Finally, I will have a form related to my dataset, collect user data and store this data in a Firebase database. The data collected will contain at least three different data types, most notably strings, integers and Booleans. I will validate this data and display a summary of the data on the form.

Technologies Used

The technologies I will use include Firebase to store and retrieve data, VS Code and Thonny to code and run my programs, matplotlib to display graphs in Python and ChartJS for the graphs on my webpage. I will be using Python for the cleaning, analsysis and basic graphs and HTML/CSS and JS for the website and interactivity. I will use LiveServer to run my website and OBSStudio and Adobe Premier Pro to record and edit the video.

Information System Chart Problem

The most major problem that I encountered through the course of my project was a visual flaw in my charts after attempting to renew them when interacted with. When interacted with the charts would be drawn again with the new parameters and the old ones would be destroyed. The issue with this method was that when hovering over the chart the old one would appear and disappear making a very visually unappealing bug.

I came to the conclusion that the chart.destroy() function was not running as intended and the old charts were not being removed from the canvas. My solution to this problem was to remake the functions in which I renew the charts to instead update them. When using the chart.update() function you can simply change the chart parameters and update the same chart without having to replace it fully solving this problem.

Analytics Algorithm

The primary function for the algorithm used in the analytics part of my project is to separate every exoplanet in the database into their respective exoplanet categories. This is done using their properties such as mass, radius and year length. Once this is done the averages of each property can be found for each category.

Pseudocode of Algorithm

.Analytics Start

.Initialise planet category dictionaries

.For each planet in exoplanet dataset

..Mass = planet earth masses

..Radius = planet earth radii

..Year length = planet orbital period in days

..If mass less than 2 and radius less than 2.5

...Add planet to Earthlike dictionary

..Else if mass less than 10 and radius less than 4

...Add planet to Super Earth dictionary

..Else if mass less than 17 and radius less than 8

...Add planet to Mega Earth dictionary

..Else if mass less than 10 and radius less than 4

...Add planet to Super Earth dictionary

..Else if year length less than 10

...Add planet to Hot Jupiter dictionary

..Else if mass less than 125

...Add planet to Neptunelike dictionary

..Else if planet jupiter masses less than 75

...Add planet to Jupiterlike dictionary

..Else add planet to Brown Dwarf dictionary
